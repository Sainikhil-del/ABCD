Build a complete, working AI-powered web application using a local LLM (Ollama) with a Retrieval-Augmented Generation (RAG) system for PDF-based question answering.

The system must be fully functional WITHOUT any paid APIs.

---

🧠 CORE FUNCTIONALITY:

* User uploads a PDF file
* System processes and stores it using embeddings
* User asks questions
* System retrieves relevant content from the PDF
* Local LLM (Ollama with Mistral) generates answers ONLY based on retrieved context

---

⚙️ BACKEND REQUIREMENTS:
Use Python.

Libraries:

* langchain
* faiss-cpu
* pypdf
* sentence-transformers

Core Implementation:

1. Load PDF using PyPDFLoader
2. Split text using RecursiveCharacterTextSplitter (chunk_size=500, overlap=50)
3. Generate embeddings using HuggingFaceEmbeddings
4. Store vectors using FAISS
5. Implement similarity search for retrieval
6. Use Ollama (model: mistral) for answer generation

IMPORTANT:

* Do NOT use OpenAI API or any paid APIs
* Everything must run locally
* Ensure clean, minimal, working pipeline

Prompt behavior:

* Answer ONLY from provided context
* If answer not found → respond: "Not found in document"
